## 1부 — 코드의 진화: v0.0.1에서 v0.0.4까지

### v0.0.1 — 콘솔 출력 (오전)

가장 단순한 버전. RSS를 4개 사이트에서 가져와 터미널에 텍스트로 출력.

**핵심 깨달음:** 인터넷의 데이터를 너의 컴퓨터로 가져오는 데이터 파이프라인의 가장 기본 형태. 이 흐름은 모든 데이터 기반 시스템의 골격이다 — 수집 → 가공 → 출력.

**한계:** 결과가 터미널에 떴다가 사라진다. 다시 보려면 매번 다시 실행해야 한다.

### v0.0.2 — HTML 파일로 저장

콘솔 출력 대신 **사람이 보기 좋은 웹페이지**를 파일로 저장하도록 변경.

**무엇이 바뀌었나:**
- 결과를 `trends_YYYYMMDD.html` 파일로 디스크에 저장
- 카드 형식의 디자인 (회색 배경, 패딩, 둥근 모서리)
- 클릭 가능한 링크
- 푸터에 *"made by ssik"* — 너의 첫 서명

**핵심 깨달음:** **"콘솔 출력"과 "파일 저장"은 다른 일이다.** 콘솔은 휘발성, 파일은 영속성. 그리고 파일을 HTML로 저장하는 순간, 그 결과물은 *너의 폴더 안의 데이터*가 아니라 *너의 브라우저에서 열리는 작은 웹사이트*가 된다. 이 차이는 사용 경험을 완전히 바꾼다.

### v0.0.3 — 마크다운 동시 저장

HTML 옆에 노션 친화적인 마크다운 파일도 같이 생성. 한 번 실행에 두 형식이 동시에 나옴.

**무엇이 바뀌었나:**
- `build_html()`과 별도로 `build_markdown()` 함수 추가
- 같은 데이터를 두 가지 형태로 변환
- 요일 표시 추가 (월·화·수…)

**핵심 깨달음:** **같은 데이터, 다른 용도, 다른 형식.** HTML은 *너가 매일 들여다보는 페이지*용이고, 마크다운은 *노션에 아카이빙*하기 위한 형식이다. 같은 일을 다양한 형태로 출력할 수 있는 능력은 진짜 도구의 조건. 마크다운은 한 번 잘 써두면 노션·블로그·GitHub README 어디든 옮겨갈 수 있는 *이식성 높은 텍스트*다.

### v0.0.4 — 요약 자동 추출

각 글의 제목 아래에 **RSS에 들어있던 description**을 꺼내 요약으로 표시.

**무엇이 바뀌었나:**
- `clean_summary()` 함수 추가 — HTML 태그를 정규표현식으로 제거하고 200자로 자르는 청소 작업
- 마크다운에서 인용구 형식(`> `)으로 요약 표시 → 노션에서 회색 인용 박스로 변환됨

**핵심 깨달음:** **데이터를 가져오는 것과, 그 데이터를 쓸 만한 형태로 정리하는 것은 별개의 일이다.** RSS는 원래 description을 보내고 있었지만, 거기엔 HTML 태그가 섞여있고 길이가 들쭉날쭉했다. *데이터 가공*이라는 단계가 그래서 필요하다. 또한 사이트마다 RSS의 description 품질이 다 다르다는 것도 직접 확인 (Hacker News는 거의 비어있고, Pitchfork는 풀 한 단락) — 이게 다음 단계(AI 요약)가 필요할지 결정하는 근거가 된다.

## 2부 — 도구가 시스템이 된 순간: 노션 아카이브

코드를 만든 것만큼 중요한 일이 오후에 있었다 — **그 코드의 결과물을 너의 일상에 끼워넣을 공간을 만든 것.**

### SSIK_HIVE의 부활

기존에 만들어둔 SSIK_HIVE 노션 페이지가 발견되었다. 패션 / 카페 / 공간 / 묵상 네 카테고리로 정리된 2024년의 페이지. 그 페이지가 보여준 것: **2년 전의 너도 지금의 너와 같은 방향을 보고 있었다.** 사람은 잘 안 변한다. 진짜 자기다움은 자기도 모르게 흔적을 남긴다.

### 새로운 구조

기존 페이지를 비우고 새로운 7-페이지 구조로 재설계:

```
🏠 SSIK_HIVE — 감각과 기술이 같이 자라는 곳
  ├─ 📌 About Me
  ├─ 🎯 This week
  ├─ 📓 Daily Log
  │    └─ Day 1 — 2026.05.25
  ├─ 🛠 Projects
  │    ├─ trend-radar
  │    └─ 📊 Daily Trends Archive
  │         └─ 2026.05.25 (월) — Day 1
  ├─ 📚 학습 노트
  ├─ 💡 Inspiration Board
  └─ 📅 Weekly Review
```

**핵심 깨달음:** 노션의 진짜 힘은 *페이지의 중첩*이다. 페이지 안에 페이지를 만들 수 있다는 것 — 이게 트리 구조다. 컴퓨터 과학에서 가장 중요한 자료구조 중 하나이고, 너는 그걸 노션이라는 도구로 자연스럽게 만지게 됐다.

### 마크다운의 첫 변환

Day 1 보고서를 노션 페이지에 붙여넣었을 때, 마크다운 문법(`#`, `**`, ` ```)이 자동으로 헤딩·볼드·코드 박스로 변환되었다. 이게 마크다운의 진짜 힘이다.

그러나 두 번째 시도(`.md` 파일을 VS Code에서 그대로 복사)에서는 노션이 *"이건 코드구나"* 하고 코드 블록으로 인식해버리는 실수가 발생. **Ctrl + Shift + V로 미리보기를 띄워 변환된 화면에서 복사**하는 방법으로 해결.

**핵심 깨달음:** **마크다운은 원본 텍스트와 변환된 화면이 다르다.** 사람이 보는 건 변환된 화면이고, 컴퓨터가 처리하는 건 원본 텍스트다. 어디서 복사하느냐에 따라 노션이 받아들이는 방식이 달라진다.

## 3부 — 자동화의 본질을 만지다

오후 후반에 *"매일 같은 코드를 돌리면 다른 결과가 나오는가?"* 라는 질문이 나왔다. 이 질문이 사실 오늘의 진짜 핵심이었다.

답은 **그렇다**. 그리고 그 *왜* 가 중요하다.

### 정적인 코드, 동적인 결과

너의 코드는 변하지 않는다. 같은 글자, 같은 함수, 같은 로직. 하지만 그 코드가 *요청할 때마다 인터넷의 현재 상태가 다르기 때문에* 결과가 달라진다.

```
코드 (변하지 않음)  +  현재 시점의 웹 상태 (매번 변함)  =  매일 다른 결과
```

이게 자동화의 본질이다. *고정된 행위를 변화하는 환경에 반복 적용해서, 매번 새로운 가치를 뽑아내는 것.* 너의 trend-radar는 그 가장 기본적인 형태다.

**핵심 깨달음:** *"인터넷의 현재 상태를 너의 화면에 비추는 작은 거울"* — 거울 자체는 안 변하고, 거울이 비추는 풍경이 매일 변한다. 그래서 같은 거울을 매일 들여다보는 일이 의미를 가진다.

## 4부 — 오늘 만진 8개 도구 정리

기술적으로 너가 오늘 처음 만져본 것들:

| 도구 | 무엇인가 | 어디서 썼나 |
|---|---|---|
| **Python** | 프로그래밍 언어 | trend-radar 전체 |
| **pip** | Python 패키지 관리자 | feedparser 설치 |
| **feedparser** | RSS를 파싱하는 외부 라이브러리 | 4개 사이트 데이터 가져오기 |
| **VS Code** | 코드 편집기(IDE) | 모든 코드 작성 |
| **터미널** | 시스템 명령어 입력하는 곳 | pip 설치, 파이썬 실행 |
| **HTML** | 웹페이지 마크업 언어 | 결과 화면 디자인 |
| **Markdown** | 가벼운 문서 포맷 | 노션용 출력, 보고서 작성 |
| **정규표현식** | 텍스트 패턴 매칭 | RSS 안의 HTML 태그 제거 |

**1학년 5월 25일, 단 하루에 8개 도구를 직접 만져본 것.** 한 학기 분량의 경험을 압축한 셈이다.

## 5부 — 시스템적으로 만든 것

낱개 도구 학습이 아니라 *시스템 하나를 끝에서 끝까지 만든 것* 이 오늘의 진짜 수확이다.

```
[수집]              [가공]              [저장]              [아카이빙]
                                                                
4개 RSS 소스   →   feedparser   →   HTML 파일       →   브라우저로 열기
                  + 정규표현식       (시각적 확인용)
                                  →   Markdown 파일   →   노션으로 복붙
                                      (아카이브용)         (Daily Trends Archive)
```

이게 작은 **데이터 파이프라인**이다. 큰 기업의 데이터 시스템도 본질은 같다 — 규모와 복잡도만 다를 뿐, 구조는 동일하다.

## 6부 — 자기 자신에 대한 작업

오늘 너가 한 가장 중요한 일은 어쩌면 코드도 노션도 아니다. **너가 두 번 멈춰서 자기 자신에게 던진 질문들이다.**

### 질문 1: *"사실 내가 한 건 복붙 아닌가?"*

이 질문이 첫 번째 보고서를 받게 만들었다. 그리고 그 보고서는 너에게 한 가지를 말해줬다 — *코드를 짜는 능력보다, 끝까지 가본 경험과 이해하려는 태도가 훨씬 귀하다.* 코드는 검색하면 나오지만, 그 두 가지는 안 나온다.

### 질문 2: *"내 작업을 아카이빙할 공간이 필요한데, 노션이 좋을까?"*

이 질문이 SSIK_HIVE의 부활을 가져왔다. 그리고 더 중요하게는 — *작업을 한 다음에 정리하는 게 아니라, 정리할 공간을 미리 만들고 시작하는 사람*이 너라는 걸 증명했다.

### 질문 3: *"매일 다른 결과가 나오는 거지?"*

이 질문이 자동화의 본질을 만지게 했다. 그리고 *"매일 돌릴 때마다 정말 다른가?"* 라는 호기심이 너를 일주일 운영의 진짜 동력으로 이어줬다.

**핵심 깨달음:** **자기 자신에게 좋은 질문을 던지는 능력**이 자기개발의 가장 중요한 무기다. 좋은 답은 좋은 질문에서만 나온다. 너는 오늘 세 개의 좋은 질문을 던졌다.

## 7부 — 일주일 운영 계획

내일부터 일요일까지 6일간, 매일 5분의 의식:

```
1. VS Code 열고 → python trend_radar.py 실행 (10초)
2. HTML 파일 더블클릭 → 브라우저로 한 번 훑어보기 (2분)
3. .md 파일 VS Code에서 열기 → Ctrl+Shift+V로 미리보기 → 복사 (30초)
4. 노션 Daily Trends Archive에 새 페이지 만들기 → 붙여넣기 (1분)
5. 페이지 맨 위에 "오늘의 한 줄" 메모 (1분)
```

이 의식의 진짜 목적은 *데이터를 모으는 것*이 아니라 **습관을 만드는 것**이다. 일주일이 쌓이면 *"트렌드의 흐름"* 이 보이기 시작하고, *"내 머릿속에서 일어난 변화"* 도 보이기 시작한다.

그리고 일요일에 첫 Weekly Review를 작성하게 된다. 7개의 점이 모여 첫 선이 되는 순간이다.
